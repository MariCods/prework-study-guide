HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • All visible elements on a webpage are postioned between two brackets. A opening and a closing tag.
  • Tags are used to orgainze and structure content on a webpage using only html. Tags include header, body,and footer repectivly.
  • For example a header tag or h# is used to orgainze the amount of headers and their postion on the page. Going from 1-6.
  • p tags: represent a paragraph or block of text.
  • li tag: is another form of orgainzation. The list tag varies with Ol or Ordered lists and Ul or Unordered lists. Basically the difference between a bulleted or numbered list.
  • img tags: are a bit different. Begining and ending with 1 open bracket. The most important quality about this element is the src which gives the location of the image
  • The struture for the img tag, the br tag, and the link tag are all the same in that the visble code is between two open brackets.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are three types of commonly used css coding practices. Inline CSS, Internal CSS, and External CSS. The later being the most common as it allows designers to keep code seperate, making the process easier.
  • Inline CSS is placed in the head section of HTML. It only applies to a page and only applies to single HTML tags mostly formated in a single line of code.
  • Internal CSS applys only to specific HTML ids and classes on a single given page.
  • link rel="stylesheet" href="./file name": is a piece of code placed in the header that connects the HTML page and the external CSS page.
  • In the CSS Box Model the content is surronded by padding, which is surrounded by a border. This border is surrounded by a margin.
  • When coding colors for selectors you can type a basic color or use hexadecimal code.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • cd: locates files in your computer directory.
  • touch: creates a file in a chosen directory.
  • a PR or pull request: is used to merge the comparison branch or in this case feature branch into the main branch for a project.
  • git pull origin main: makes sure the files are in sync between the local computer and the git repo.
  • git push origin "file name": Is pushing the local file changes to the remote git repo.
  • mkdir: creates a new directory.
